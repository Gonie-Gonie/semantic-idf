Start

IDF Analyzer opens EnergyPlus IDF, IMF, and epJSON-style files, shows the input in synchronized Text, JSON, and Table views, and calculates Summary, Diagnose, and Geometry results in the right panel. A reference office model loads at startup so the interface is usable before you open a project file; analysis starts after the first screen is visible and fills in the visible Summary first.

Files

Open

Loads `.idf`, `.imf`, `.epjson`, `.json`, or `.txt` from the desktop file picker and analyzes it automatically.

Save

Writes the current editor text back to the opened file. If there is no original file path, the app asks where to save.

Revert

Restores the input text to the snapshot from the most recent open operation.

Automatic Analysis

Analysis runs after opening a file and after editor changes pause briefly, so repeated edits do not need a separate Analyze button.

CLI Commands

The packaged executable can also run non-interactive commands. Running without arguments opens the desktop app; running `idf-analyzer cli ...` or a recognized top-level command runs the CLI. Use `-` as the input path to read from stdin and `-o -` to write command output to stdout.

Summary and diagnostics

Export text, JSON, CSV, or styled XLSX summaries, and export Diagnose issues for scripts or CI checks.

idf-analyzer cli summary model.idf
idf-analyzer cli summary -format json -o summary.json model.idf
idf-analyzer cli summary -format csv -o summary.csv model.idf
idf-analyzer cli summary -format xlsx -o summary.xlsx model.idf
idf-analyzer cli diagnostics -format text model.idf
idf-analyzer cli diagnostics -format csv -o diagnostics.csv model.idf
idf-analyzer cli analyze -format json -o report.json model.idf

Multi-file summary

Compare Summary metrics across several models as CSV, JSON, text, or a styled single-sheet XLSX workbook.

idf-analyzer cli multi-summary -format csv -o compare.csv a.idf b.idf c.epjson
idf-analyzer cli multi-summary -format xlsx -orientation metrics -o compare.xlsx a.idf b.idf
idf-analyzer cli multi-summary -format xlsx -orientation files -o compare-by-file.xlsx a.idf b.idf

Clean

Preview cleanup candidates, apply default or explicit rules, compact formatting, and optionally rename duplicate semantic object names.

idf-analyzer cli clean --dry-run model.idf
idf-analyzer cli clean --dry-run -format json -o cleanup-preview.json model.idf
idf-analyzer cli clean -o cleaned.idf model.idf
idf-analyzer cli clean -rules all --compact -o cleaned.idf model.idf
idf-analyzer cli clean -rules none --semantic-duplicates -o semantic-fixed.idf model.idf
idf-analyzer cli clean -rules remove_unused_schedules,remove_duplicate_output_variables -o cleaned.idf model.idf

Convert

Convert IDF and epJSON inputs to IDF, epJSON, semantic YAML view export, or a styled one-sheet Excel table export grouped by `[ObjectType]` markers.

idf-analyzer cli convert -to idf -o model.idf model.epjson
idf-analyzer cli convert -to json -o model.epjson model.idf
idf-analyzer cli convert -to yaml -o model.semantic.yaml model.idf
idf-analyzer cli convert -to table -o model.tables.xlsx model.idf
type model.idf | idf-analyzer cli convert -to json -o - -

Input Views

Text

Shows editable object summaries above the raw source text. The raw text area remains the final source that will be saved.

JSON

Shows an epJSON-shaped structured view. Value tokens can be edited inline while syntax tokens remain read-only.

Table

Shows object-type tables with switchable row and column orientation for scanning fields across similar objects.

Filter

The left-panel filter works the same way across Text, JSON, and Table views by matching object type, object name, field label, and value text.

Summary

Summary groups model inventory, area, envelope, load, schedule, and HVAC metrics. Use the Summary filter to narrow the metric list. Export JSON keeps the grouped structure, while Export CSV creates `name,value` rows. CSV names use variable-style names with units in brackets, including `[-]` for values without a physical unit.

Diagnose

Diagnose lists likely run-blocking errors and warning-level modeling risks, including missing references, duplicate names, orphan resources, missing required objects, geometry issues, unsupported schedule-hour calculations, and disconnected HVAC node graph hints.

Geometry

Geometry shows parsed zones, surfaces, and fenestration in 3D and plan views. Selecting an item shows key metrics and related objects such as parent zones, base surfaces, child windows, adjacent zones, and matching surfaces. Sync locate can move the input view to the selected model object.

Batch Summary

Batch Summary is available from Batch. Choose it from the left selector, select several input files, wait for the progress bar to finish, then review the combined Summary table. Metrics can be shown as rows with buildings as columns, or transposed with buildings as rows. Export CSV uses the currently selected table direction.

Diagnose / Fixes

Cleanup suggestions are available from Diagnose and work on the input currently open in the main app. Choose rules on the left, then review filterable candidate checkboxes on the right to include or exclude individual removals. Apply selected updates the editor and re-runs analysis, while Save cleaned copy writes a new file. Current rules cover unused schedules, unused envelope resources, unused curves/tables, duplicate Output objects, and compact formatting. Comment-only cleanup is reserved for later.

Settings

Settings opens as its own page. Appearance controls choose the app theme and Geometry colors for zones, walls, roofs, windows, selections, and the 3D background. Behavior controls store defaults for automatic analysis delay, raw text sync, and Geometry sync locate.
